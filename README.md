# JWT Tool v1.0

Burp Suite 扩展插件，用于 JWT 令牌的解析、编码、验证和爆破攻击。

## 工具截图



## 功能特性

### 1. JWT 解析与解码
- 自动解析 JWT 令牌并显示 Header 和 Payload
- 支持 JSON 格式化显示，方便阅读
- 自动识别算法类型（HS256、HS384、HS512、RS256、ES256 等）

### 2. JWT 编码与签名
- 支持手动编辑 Header 和 Payload
- 支持多种签名算法：
  - HMAC: HS256、HS384、HS512
  - RSA: RS256、RS384、RS512
  - RSA-PSS: PS256、PS384、PS512
  - ECDSA: ES256、ES384、ES512
  - EdDSA
- 填入密钥后自动生成签名

### 3. 签名验证
- 输入密钥验证 JWT 签名是否有效

### 4. 密钥爆破
- 支持字典爆破 HS 系列算法
- 支持多种编码类型：
  - None: 原始密钥
  - Base64: Base64 编码
  - MD5: MD5 哈希
  - MD5_16: MD5 哈希前16位
  - ALL: 遍历所有编码类型
- 内置默认字典（释放到插件目录）
- 支持自定义字典
- 实时显示爆破进度

### 5. HTTP 请求提取
- 支持从 Burp 代理历史记录中提取 JWT
- 自动识别以下格式：
  - Bearer Token
  - token 参数
  - Authorization header
  - 通用 JWT 格式

### 详细组件

#### 第一列：数据包列表
- 显示通过右键菜单发送的 HTTP 请求列表
- 选中后可查看请求中的 JWT

#### 第二列：JWT 输入
- JWT 输入框：输入待处理的 JWT 令牌
- 预留区域：用于其他功能扩展

#### 第三列：操作按钮
- **解码 》**: 将 JWT 解码为 Header 和 Payload
- **编码 《**: 将编辑后的 Header、Payload 和密钥重新编码为 JWT
- **校验**: 验证 JWT 签名是否有效

#### 第四列：JWT 解析
- **Header**: 显示 JWT 头部信息（格式化 JSON）
- **Payload**: 显示 JWT 载荷信息（格式化 JSON）
- **Verify**: 显示当前 JWT 的签名
- **Secret**: 输入密钥用于签名或验证

#### 爆破设置
- **编码类型**: 选择密钥编码方式
- **开始爆破**: 开始密钥爆破
- **停止**: 停止爆破
- **字典路径**: 显示当前使用的字典路径
- **选择字典**: 选择自定义字典文件
- **进度条**: 显示爆破进度

## 使用方法

### 安装
1. 打开 Burp Suite
2. 进入 Extender（扩展）-> Extensions（已安装扩展）
3. 点击 Add（添加）
4. 选择 JWT_Tool_v1.0.jar 文件
5. 加载成功后会在 Burp 顶部看到 "JWT Tool" 标签页

### 基本使用流程

#### 1. 解码 JWT
在 JWT 输入框中粘贴 JWT 令牌，点击 **解码 》** 按钮，Header 和 Payload 会显示在右侧。

#### 2. 修改并重新编码
1. 修改 Header 或 Payload 内容
2. 在 Secret 框中输入密钥
3. 点击 **编码 《** 生成新的 JWT

#### 3. 验证签名
1. 输入待验证的 JWT
2. 点击 **解码 》** 解析 JWT
3. 在 Secret 框中输入密钥
4. 点击 **校验** 按钮验证签名

#### 4. 密钥爆破
1. 输入待爆破的 JWT
2. 点击 **解码 》** 解析 JWT
3. 选择编码类型（建议选择 ALL）
4. 确保字典已加载（内置字典会自动加载）
5. 点击 **开始爆破**
6. 找到密钥后会自动填入 Secret 框

### 从 HTTP 请求中提取 JWT
1. 在浏览器中触发包含 JWT 的请求
2. 在 Burp 代理历史记录中找到该请求
3. 右键点击请求
4. 选择 "Send to JWT Tool"
5. JWT 会自动填入并解析

## 字典说明

### 内置字典
插件内置了常用的弱口令字典，首次加载时会自动释放到插件同目录的 `wordlist.txt` 文件中。

### 自定义字典
点击 **选择字典** 按钮选择自定义字典文件，字典文件格式为每行一个密钥。

### 编码类型
- **None**: 不编码，直接使用原始密钥
- **Base64**: 对密钥进行 Base64 编码
- **MD5**: 对密钥进行 MD5 哈希
- **MD5_16**: 对密钥进行 MD5 哈希并取前16位
- **ALL**: 遍历所有编码类型组合

## 注意事项

1. **算法限制**: 密钥爆破功能仅支持 HS256、HS384、HS512 等 HMAC 对称算法
2. **非对称算法**: RS、ES、PS、EdDSA 等非对称算法无法通过字典爆破
3. **字典路径**: 内置字典会在首次加载时释放到插件同目录
4. **字典编码**: 如果密钥在字典中是编码后的值，选择对应的编码类型
